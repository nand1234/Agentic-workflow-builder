# Design Reviews

This directory contains design analysis docs generated by /review-design.

Each file is named after the feature slug:
- discount-codes.md
- user-auth.md
- password-reset.md

## How files get here

Run /review-design with a Figma URL, PNG paths or both:

```bash
# PNG only
/review-design discount-codes ./designs/checkout-flow.png

# Figma URL only
/review-design discount-codes https://www.figma.com/file/ABC123/Checkout

# Both
/review-design discount-codes https://www.figma.com/file/ABC123/Checkout ./designs/edge-cases.png
```

## Figma Token Setup

For richer Figma API data (component tree, tokens, metadata):

```bash
export FIGMA_TOKEN=your_personal_access_token
```

Get your token at: https://www.figma.com/settings → Personal access tokens

Without a token, /review-design falls back to visual analysis of the Figma URL
and works perfectly well for most use cases.

## How these docs are used

- /discover reads design docs to inform requirements and BDD criteria
- /plan reads design docs to break frontend work into accurate tickets
- /implement reads design docs to build against the exact spec
- frontend-engineer agent preloads design-context which references these docs
