# Agentic Workflow Builder

A collection of Claude Code skills, agents, commands, and prompt templates for building agentic engineering and QA workflows. Each top-level folder is an independent, drop-in workflow you can load into Claude Code.

---

## Repository Layout

| Folder / File | What it is |
|---|---|
| [`ai-dev-worflow/`](#ai-dev-worflow) | End-to-end feature development workflow (discover → plan → implement → verify → merge) using skills, subagents, and worktrees |
| [`qa-workflow/`](#qa-workflow) | "GET TESTS DONE" — two-mode QA automation system covering E2E, API, load, security, and performance |
| [`ad-hoc-testing/`](#ad-hoc-testing) | Lightweight `playwright-cli` skill for ad-hoc browser automation, snapshots, and data extraction |
| [`repo-mindmap/`](#repo-mindmap) | CodeMap orchestrator — analyses a repo, builds an architecture graph, and renders a React visualiser |
| [`Prompts/`](#prompts) | Standalone system prompts (currently: Agentic QA prompt for diff-aware test generation) |
| [`output/`](#output) | Example outputs generated by the FlowForge skill (`hey_brother_flow`, `how-are-you-reply`) |
| [`python-flow-generator-SKILL.md`](#flowforge-python-flow-generator) | FlowForge skill — turns plain-English workflows into tested, standalone Python scripts |
| [`Workflow-builder-template.md`](#workflow-builder-template) | Deterministic workflow code generator (v6.0) — language-agnostic, no AI at runtime |
| [`Engineering-workflow-template.md`](#engineering-workflow-template) | Master strategic multi-agent orchestrator template for TDD-verified bug RCA and impact analysis |

## ai-dev-worflow

Full Claude Code workflow system for shipping features. Combines forked subagents (context isolation) with an Opus advisor (intelligence escalation).

**Onboarding (once per project):**
```bash
/enrich-business     # back business workflows with code references
/scan-frontend       # generate frontend architecture doc
/scan-backend        # generate backend architecture doc
```

**Per feature:**
```bash
/review-design $FEATURE [figma-url|png]
/discover $FEATURE
/plan $FEATURE
/setup-worktrees $FEATURE
/next-ticket $FEATURE
/implement $TICKET $FEATURE
/verify-coverage $TICKET $FEATURE
/merge-to-cp $TICKET $FEATURE
/sync-docs $FEATURE
```

**Includes:**
- 11 specialist agents under `.claude/agents/` (logic-explorer, requirements-analyst, bdd-writer, tech-planner, frontend/backend-engineer, pr-reviewer, design-reviewer, devil-advocate, framework-scanner, coverage-verifier)
- 23 skills under `.claude/skills/` covering discovery, planning, implementation, review, recovery (`rollback-ticket`, `revise-plan`), and silent context loaders
- Quality gates: 100% unit coverage, RTM fully mapped, agents hard-blocked from `main`/`master`

See [`ai-dev-worflow/README.md`](ai-dev-worflow/README.md) and [`ai-dev-worflow/CLAUDE.md`](ai-dev-worflow/CLAUDE.md) for the full guide.

## qa-workflow

QA automation system with two modes:

- **Mode 1 — Quick:** `/qa:discover` once, then `/qa:test [description]` for everything else.
- **Mode 2 — Structured:** `/qa:strategy` → `/qa:plan-{e2e,load,security,performance}` → `/qa:execute` → `/qa:run` → `/qa:verify` → `/qa:report`.

**Includes:**
- Commands under `.claude/commands/qa/` (discover, test, strategy, plan, execute, run, verify, report, coverage-gap, maintain, migrate, quick, update_skill, help)
- Specialist agents (`qa-researcher`, `specialist-agents`)
- Framework standards skills: `playwright-standards`, `cypress-standards` (SOLID, POM, DRY, banned `waitForTimeout`, pre-commit checklist)
- User guide at `qa-workflow/.claude/docs/USER-GUIDE.md`

See [`qa-workflow/.claude/commands/qa/README.md`](qa-workflow/.claude/commands/qa/README.md) for the full command reference.

## repo-mindmap

CodeMap orchestrator that takes a repo path and produces an architecture mindmap.

- `SKILL.md` — orchestrator (entry: `analyserepo /path/to/project`)
- `architecture_SKILL.md` — Skill 0: architecture discovery
- `file-analysis_SKILL.md` — Skill 1: per-file deep analysis
- `graph-synthesis_SKILL.md` — Skill 2: graph synthesis
- `visualiser-template.jsx` — React visualiser template
- Output is written to `.codemap/` inside the analysed project (`session.json`, `files.json`, `arch.json`, `graphs.json`, `files/{name}.json`)

## FlowForge (python-flow-generator)

`python-flow-generator-SKILL.md` defines the `pythonflowforge` skill. Describe a workflow in plain English; Claude Code plans, codes, tests, fixes, and packages it into a folder of step scripts plus a single `execute_flow.py` orchestrator. The output runs with **zero AI dependency at runtime**.

Phases: clarify → plan → context schema → step contracts → write step scripts → write orchestrator → test/fix loop → package.

## Workflow-builder-template

`Workflow-builder-template.md` (v6.0) — deterministic, language-agnostic workflow code generator. Asks for language, API client, and output path, then writes a complete runnable project under `./output/<workflow-name>/`. No PLAN.json, no AGENT.md, no AI runtime unless explicitly requested.

---

## Engineering-workflow-template

`Engineering-workflow-template.md` — master strategic multi-agent orchestrator for TDD-verified bug RCA and impact analysis. Defines a permanent memory tree under `.claude/memory/` (architect, frontend, backend, db_schema, logic, linker.json) and a routing engine that dispatches sub-agents across the FE → BE → DB stack.

## Getting Started

1. Pick the workflow that matches your task (feature dev, QA, repo analysis, ad-hoc testing, or generic flow generation).
2. Open Claude Code in that folder — its `.claude/` config is auto-loaded.
3. Run the entry command listed above for that workflow.

For the standalone skill files at the repo root (`python-flow-generator-SKILL.md`, `Workflow-builder-template.md`, `Engineering-workflow-template.md`), point Claude Code at the file or paste its contents to load the skill.
